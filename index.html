📚 사용자 가이드

WebSquare Publishing Editor란?

절대좌표 기반 WebSquare XML을 상대좌표(반응형 가능) XML로 자동 변환하는 브라우저 도구입니다. 코드 지식 없이 파일을 드래그&드롭하면 변환·검증·미리보기까지 한 번에 처리됩니다.

3개 탭 한눈에 보기

탭언제 사용핵심 출력
XML to XML (기본)일반 변환 작업 대부분상대좌표 XML *_rel_v1.xml + HTML 미리보기/저장 *_rel_v1.html
XML 좌표변환레거시 규칙 기반 변환과 비교 시규칙 기반 변환 XML + Wireframe MD
배치 비교변환 전/후를 실 서버에서 일괄 검증스크린샷 + 보존율 리포트

처음 켰을 때 추천 순서

  1. 상단 설정(⚙)에서 저장 위치 폴더를 지정하세요. (지정 안 하면 XML은 다운로드 폴더, HTML은 preview/ 기본 사용)
  2. XML to XML 탭에서 절대좌표 XML 파일을 드롭하여 변환을 체험합니다.
  3. 결과의 와이어프레임과 검증 탭에서 누락이 없는지 확인합니다.
  4. HTML 미리보기 탭에서 실제 렌더 모습을 시각적으로 확인하고, 필요하면 HTML 저장으로 정적 HTML 파일로 추출합니다.
  5. 여러 파일을 한 번에 처리하려면 폴더 선택 & 일괄 변환 사용 (옵션으로 HTML 동봉 생성 가능).
  6. 실 서버에서 결과를 확인하려면 배치 비교 탭으로 이동 (별도 캡처 서비스 필요 — 아래 안내 참고).

상단 공용 버튼

  • ▶ 화면 비교하기: compare.html 별도 창. 두 XML을 좌우로 띄워 단순 비교.
  • 📚 사용자가이드: 지금 보고 있는 이 창.
  • ⚙ 설정: 변환 결과 저장 위치 지정.
  • 초기화: 현재 로드된 파일·변환 결과 비움 (저장된 파일은 그대로).

XML to XML 탭 — 핵심 변환 기능

실제 변환 샘플 쌍에서 추출한 패턴으로 변환합니다. 대부분의 작업은 이 탭에서 진행합니다.

1) 단일/다중 파일 변환

  1. 좌측 드롭존에 절대좌표 XML 파일을 드래그&드롭 (여러 개 동시 가능)
  2. 상단 옵션에서 일반 변환 또는 반응형 변환 선택
  3. 자동으로 변환되며 우측 탭에서 결과 확인
  4. 복사 또는 다운로드로 결과 저장
일반 vs 반응형: 반응형을 선택하면 변환된 테이블에 adaptive="layout" adaptiveThreshold="768" 속성이 추가되어 768px 미만에서 자동으로 모바일 레이아웃이 됩니다. 그 외 구조·내용은 완전히 동일합니다.

2) 폴더 일괄 변환

  1. 탭 상단 폴더 선택 & 일괄 변환 버튼 클릭
  2. 변환할 폴더 선택 (하위 폴더까지 재귀 스캔)
  3. 같은 폴더에 _rel_v*.xml 쌍이 없는 원본 XML만 자동 변환
  4. 결과는 원본 옆에 <name>_rel_v1.xml로 저장됨
  5. HTML도 생성 (.html) 체크박스를 켜면 XML 옆에 <name>_rel_v1.html도 동봉 출력
  6. 진행률은 오버레이로 표시. 실패한 파일은 F12 콘솔에서 확인
주의: File System Access API를 사용하므로 Chrome / Edge 등 Chromium 계열 브라우저에서만 동작합니다. Firefox/Safari에서는 일괄 변환 버튼이 동작하지 않습니다.
HTML 동봉 생성 시: HTML 파일의 CSS 링크는 css/base.css 같은 상대경로로 박힙니다. 저장 폴더 옆에 프로젝트의 css/ 와 images/ 폴더가 같이 있어야 정상 렌더됩니다. (디자인 깨짐 시 FAQ 참고)

3) 우측 결과 패널 활용

탭내용
변환 XML최종 결과 텍스트. 복사·다운로드 가능
절대좌표 와이어프레임원본 좌표 기반 시각화 (GroupBox 타이틀 표시)
상대좌표 와이어프레임변환 결과 구조 시각화. 라벨/ID 토글, 셀ID 토글 지원
HTML 미리보기WebSquare base.css/product.css 적용된 실제 렌더 미리보기 (iframe). 상단 토글: WS 속성 보존(data-ws-*) / CSS 인라인
검증컴포넌트 통계, 겹침/좌표 이상, 스크립트 핸들러, ID 누락 등 자동 점검
ID 불일치 빨간색 표시: 상대좌표 와이어프레임에서 빨간 ID는 원본에 있던 ID가 변환 후 누락된 것입니다. 숨김 컴포넌트는 자동 복구되지만, 화면 표시 컴포넌트가 빠진 경우는 검증 탭의 경고를 따라 수동 확인하세요.

4) 검증 결과 보는 법

  • PASS — 문제 없음
  • FAIL — 항목별로 구체적 오류 설명과 영향이 함께 표시됨
  • 이벤트 핸들러(scwin.{id}_OnClick), {id}.Value 참조, ref/dataList 바인딩, disabled/maxlength 등 속성 보존을 모두 검사

5) HTML 변환·미리보기·저장 (신규)

변환된 상대좌표 XML을 정적 HTML 파일로 추출합니다. WebSquare 런타임 없이 브라우저에서 바로 열어 디자인/구조를 확인할 수 있습니다. 내부적으로 js/xml-to-html.js 모듈이 xf:input/xf:select1/w2:textbox 등을 표준 HTML 태그(input/select/span)로 치환합니다.

미리보기 옵션 (HTML 미리보기 탭 상단)

  • WS 속성 보존 (data-ws-*): 켜면 ctype/hierarchy/orgid/ref 등 WebSquare 전용 속성을 data-ws-*로 변환해 보존. 디버깅/추적 시 유용. 끄면 표준 HTML 속성만 출력
  • CSS 인라인: 켜면 base.css/product.css/bridge.css 내용을 <style>로 임베드해서 자기완결적 HTML 생성 (단독 실행 가능, 파일 크기 큼). 끄면 <link rel=stylesheet> 외부 참조

저장 위치 — 기본은 preview/ 폴더

  • 액션 바의 HTML 저장 버튼을 누르면 첫 사용 시 폴더 선택 dialog가 뜹니다. 프로젝트 루트의 preview/ 폴더를 선택하세요.
  • 한 번 선택하면 IndexedDB에 기억되어 다음부터는 자동 사용됩니다 (재선택 불필요).
  • 다른 위치로 저장하려면 설정(⚙)에서 저장 폴더를 지정하세요 — 거기가 우선됩니다 (XML/HTML 모두).
  • 저장 시 CSS 경로는 자동 조정: preview/에 저장 → ../css/..., 사용자 지정 폴더 → css/... 유지

적용되는 CSS 스택

파일역할
css/base.cssWebSquare 핵심 스타일 (.w2input/.w2tb/.btn_cm 등)
css/product.css제품 레벨 스킨/색상 (KB 매핑된 클래스 포함)
css/bridge.css변환기 호환 레이어 — 표준 HTML 태그 ↔ WebSquare 클래스 갭 메움. vendor CSS는 손대지 않음
"HTML 복사" 버튼은 현재 활성 파일의 HTML 텍스트를 클립보드에 복사합니다. 외부 도구로 빠르게 옮길 때 활용하세요.

XML 좌표변환 탭 (레거시)

skill/convert-xml.md의 규칙을 JS에 하드코딩한 구식 변환기입니다. 기본 작업은 XML to XML 탭을 사용하고, 이 탭은 결과 비교·검증 용도로만 사용하세요.

XML to XML과의 차이

항목XML 좌표변환 (레거시)XML to XML (권장)
변환 근거MD 규칙 하드코딩실 샘플 쌍에서 추출한 패턴

사용 방법

  1. 좌측 드롭존에 XML 파일 업로드
  2. 자동으로 규칙 기반 변환 수행
  3. 변환 XML / Wireframe MD 탭에서 결과 확인
  4. HTML → XML 역변환도 같은 탭에서 가능 (HTML 파일을 드롭하면 자동 인식)

배치 비교 탭 — 실서버 렌더 검증

변환 전/후 XML을 실제 WebSquare 서버에서 렌더하여 스크린샷·컴포넌트 카운트로 보존율을 자동 산출합니다. 많은 화면을 한꺼번에 검증할 때 사용합니다.

사전 준비 (최초 1회)

  1. 탐색기에서 프로젝트 폴더의 tools 폴더로 이동
  2. 커맨드 창을 열어 npm install 실행 (Puppeteer + Chrome ~170MB 다운로드)
  3. 설치 완료 후 프로젝트 루트의 start-editor.bat 더블클릭
  4. 캡처 서비스 창(:5678)과 에디터가 함께 뜨면 준비 완료
중요: 배치 비교는 캡처 서비스가 실행 중이어야만 동작합니다. 이후 사용 시에도 매번 start-editor.bat으로 켜주세요.

설정 항목

항목설명
WebSquare 서버172.16.0.10:8080 형식. http/https 자동 인식
서버 루트 폴더URL 경로의 시작점 (서버에서 화면 XML이 위치한 공통 상위)
비교 대상 폴더서버 루트 하위 경로. w2xPath가 자동 계산됨
리포트 폴더스크린샷·HTML이 저장될 로컬 폴더
엔트리 파일명리포트 첫 페이지 파일명 (보통 index.html)
캡처 대기(ms)렌더 안정화 대기 시간. 화면이 무거우면 3000~5000 권장
뷰포트(W×H)캡처 해상도

실행 순서

  1. 설정 완료 후 파일 쌍 스캔 — foo.xml ↔ foo_rel_v*.xml 자동 매칭
  2. 미변환 파일이 있으면 미변환 자동 변환 버튼으로 일반/반응형 옵션 지정 후 일괄 변환 가능
  3. 비교할 쌍을 체크박스로 선택
  4. 실행 클릭 — 원본·변환 병렬 캡처. 진행률 표시, 중간 중단 지원
  5. 완료 후 리포트 폴더의 엔트리 HTML을 탐색기에서 직접 더블클릭
"엔트리 열기" 버튼은 요약 확인용: 에디터의 엔트리 열기 버튼은 blob URL을 사용해서 chunk 페이지 상대링크가 깨집니다. 전체 리포트를 보려면 반드시 탐색기에서 엔트리 HTML 더블클릭하세요.

보존율 산출 방식

  • 모든 frame에서 카테고리별(폼/버튼/링크/이미지/테이블/그리드/탭/패널) 가시 컴포넌트 수와 보이는 텍스트 토큰을 추출
  • 카테고리별 보존율 = min(원본, 변환) / max(원본, 변환)
  • 텍스트 = Jaccard 유사도
  • 종합 보존율 = 양쪽 중 하나라도 비영인 카테고리들의 동일 가중 평균

리포트 구조

{리포트폴더}/
  {엔트리파일}            ← run 목록·요약·chunk 링크
  manifest.json           ← 전체 메타데이터
  runs/{runId}/
    chunk_NNN.html        ← 화면 100개 단위 비교 페이지
    images/
      NNNN_orig.jpg       ← 원본 스크린샷
      NNNN_conv.jpg       ← 변환 스크린샷

각 화면 단위의 체크박스·메모는 localStorage에 자동 저장되어 페이지를 새로 열어도 유지됩니다.

자주 묻는 질문

Q. 변환 결과 파일이 저장 폴더에 안 보여요

상단 ⚙ 설정에서 저장 위치를 지정하지 않으면 XML은 브라우저 기본 다운로드 폴더로, HTML은 preview/ 폴더로 갑니다. 명시적 저장 위치를 원하시면 설정에서 폴더를 선택하세요. 권한은 브라우저가 한 번만 묻고 이후 자동 기억됩니다.

Q. HTML 저장은 어디로 떨어지나요?

  • 설정에서 저장 폴더 지정함 → 거기 (XML/HTML 동일 위치)
  • 지정 안 함 → preview/ 폴더 (프로젝트 루트). 첫 사용 시 한 번 폴더 선택 dialog가 뜨고, 이후 자동 기억
  • 둘 다 안 되거나 권한 거부 → 브라우저 기본 다운로드 폴더로 폴백

Q. preview/ 폴더는 뭔가요?

HTML 변환 결과의 기본 저장 위치입니다. 프로젝트 루트의 css/ · images/ 와 같은 레벨에 있어, 저장된 HTML이 ../css/base.css 형식으로 상대경로 참조해 정상 렌더됩니다. 폴더가 없으면 직접 만들거나 첫 HTML 저장 시 폴더 선택 dialog에서 새로 생성하세요.

Q. 저장한 HTML 파일을 열었는데 디자인이 깨져요

  • CSS 경로 확인: HTML 파일이 있는 위치를 기준으로 css/base.css 또는 ../css/base.css 가 실제로 존재해야 합니다.
  • preview/ 에 저장한 경우: 자동으로 ../css/ 경로가 박힙니다. 프로젝트 루트의 css/ · images/ 폴더가 그대로 있어야 합니다.
  • 임의 폴더에 저장한 경우: HTML 옆에 css/ 와 images/ 폴더를 같이 복사해두거나, 미리보기 옵션의 CSS 인라인을 켜서 저장하면 단독 실행 가능
  • file:// 보안 차단: 일부 브라우저는 file://에서 SVG 아이콘 로드를 막을 수 있음. 그래도 레이아웃은 보임

Q. HTML 미리보기에서 입력 박스/버튼이 어긋나 보입니다

bridge.css가 inline-flow의 vertical-align: middle로 정렬을 통일합니다. 그래도 어긋나면 F12 → 개발자도구 → Elements로 해당 요소의 computed style을 확인하고, bridge.css에서 보강이 필요하면 알려주세요.

Q. 변환된 HTML에 data-ws-* 속성이 잔뜩 붙어 있어요

WebSquare 전용 속성(ctype/hierarchy/orgid/ref 등)을 추적·디버깅을 위해 data-ws-*로 보존한 것입니다. HTML 미리보기 탭 상단의 WS 속성 보존 토글을 끄면 표준 HTML 속성만 남깁니다 (출력이 더 깔끔해짐).

Q. 폴더 일괄 변환 버튼이 동작하지 않아요

File System Access API는 Chrome / Edge / 기타 Chromium 계열 브라우저에서만 동작합니다. Firefox·Safari에서는 단일 파일 변환만 사용하세요.

Q. 배치 비교 탭에서 캡처가 진행되지 않습니다

  • 캡처 서비스가 켜져 있는지 확인 (start-editor.bat으로 기동)
  • http://localhost:5678/health에 접속해서 응답이 오는지 확인
  • WebSquare 서버 주소가 현재 네트워크에서 실제로 접근 가능한지 확인
  • 화면이 무거우면 캡처 대기(ms)를 3000~5000으로 늘려보세요

Q. 변환된 XML에 adaptive="layout"이 자꾸 붙어요 / 안 붙어요

변환 옵션의 일반 / 반응형 선택에 따라 결정됩니다. 반응형 선택 시 w2tb tbl 테이블 그룹에 adaptive="layout" adaptiveThreshold="768"이 추가됩니다. 원하지 않으면 일반 변환으로 재변환하세요.

Q. 변환 후에 ID가 빨갛게 표시됩니다

원본에 있던 ID 일부가 변환 결과에서 누락됐다는 뜻입니다. 숨김 컴포넌트(display:none)는 자동 복구되지만, 화면에 보이는 컴포넌트가 누락된 경우는 검증 탭의 경고 메시지를 확인하고 수동으로 확인하세요.

Q. 검색영역(schbox)으로 잘못 묶이거나 안 묶입니다

검색영역 판별은 다음 조건을 모두 만족해야 합니다.

  • 화면의 첫 번째 GroupBox
  • 폼 요소 포함 / 그리드 없음
  • 버튼 텍스트가 정확히 "조회" / "검색" / "초기화" (또는 화면 우측 60% 이상 위치)

"상품조회"처럼 다른 글자가 포함된 버튼은 검색버튼으로 인식되지 않습니다. 원본 화면을 위 조건에 맞게 정리해야 합니다.

Q. 변환 결과의 컴포넌트 순서가 원본과 달라 보입니다

섹션은 절대좌표 top 기준으로 정렬됩니다. 같은 Y 좌표에 있는 GroupBox들이 left 50px 이상 떨어져 있으면 자동으로 .lybox > .col_N 좌우 분할이 적용됩니다. 의도와 다르다면 원본의 좌표를 확인하세요.

Q. 에디터가 갑자기 느려졌어요

큰 폴더를 일괄 변환 중이거나, 한 번에 너무 많은 파일을 로드한 경우입니다. 상단의 초기화 버튼으로 메모리에 로드된 파일을 비우면 회복됩니다.
