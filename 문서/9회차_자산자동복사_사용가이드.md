# 폴더 일괄변환 — 자산 자동복사 (9회차 추가)

## 개요
폴더 일괄변환(XtX 탭) 실행 시, 원본 폴더의 `css/` 와 `images/` 하위 파일을
저장 폴더에 동일 구조로 자동 복사합니다.

## 사용 방법
1. 상단 탭에서 **XtX** 선택
2. **css/ images/ 함께 복사** 체크박스 확인 (기본 ON)
3. **폴더 선택 & 일괄 변환** 클릭
4. 원본 폴더 선택 → 저장 폴더 선택
5. 진행률 완료 후, 저장 폴더 안에 `css/` `images/` 가 생성된 것을 확인

## 동작 규칙
- `css/` 또는 `images/` 가 원본에 **없으면 silently skip** (변환은 정상 진행)
- 같은 이름 파일이 저장 폴더에 이미 있으면 **덮어쓰기**
- 복사 실패해도 변환 자체는 중단되지 않음 (콘솔에 경고만 출력)
- 취소(Cancel) 버튼 누르면 진행 중인 자산 복사도 함께 중단

## FAQ
**Q. 토글 OFF 하면 어떻게 되나요?**
A. 자산 복사를 건너뛰고 XML/HTML 변환만 수행합니다.

**Q. css/ 안에 하위 폴더가 있으면?**
A. 9회차 범위는 1-depth 만 지원. 하위 폴더는 다음 회차에서 검토.

**Q. 콘솔의 `[asset-copy] N/M` 가 의미하는 것은?**
A. `N` = 실제 복사된 파일 수, `M` = 복사 계획 총 파일 수.

## 스모크 테스트 절차 (사용자 직접 수행)

자동 회귀 불가 — 브라우저 FileSystemAccess API 의존이라 node 환경에서 검증 불가능.

다음 3 케이스를 브라우저에서 수동 확인:

| # | 입력 | 기대 동작 | 콘솔 메시지 |
|---|------|-----------|--------------|
| 1 | css/ + images/ 있는 폴더 | 저장 폴더에 css/ images/ 생성 + 파일 복사 | `[asset-copy] N/N 복사 완료` |
| 2 | css/ images/ 없는 폴더 | 변환만 수행, 자산 폴더 생성 안 함 | `[asset-copy] 0/0 복사 완료` 또는 무메시지 |
| 3 | 토글 OFF | 변환만 수행, 자산 폴더 생성 안 함 | (asset-copy 메시지 없음) |
